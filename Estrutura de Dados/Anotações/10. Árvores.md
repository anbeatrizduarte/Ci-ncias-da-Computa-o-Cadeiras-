
#### Sobre

Uma árvore é uma estrutura de dados não linear que organiza os dados de forma hierárquica. Diferente de listas e array, onde a ordem é sequencial, aqui é a relação de "quem está abaixo de quem".

Podemos definir também como um conjunto finito de elementos denominados nós ou vértices tais que:

- T = vazio. e a árvore é dita vazia, ou
- existe um nó especial chamado raiz de T(r(T)); os restantes constituem um único conjunto vazio ou são divididos em m>=1 conjuntos disjuntos não vazios. as subárvores de r(T), ou simplesmente subárvores, cada qual, por sua vez, uma árvore.

Algumas outras definições fundamentais:


| Termo    | Definição                                                      |
| -------- | -------------------------------------------------------------- |
| Raiz     | O nó mais alto da árvore. O que não possui pai                 |
| Nó pai   | Um nó que está conectado diretamente acima de outro nó         |
| Nó filho | Um nó conectado diretamente abaixo de outro nó                 |
| Nó folha | Um nó que não possui filhos. É um "fim de linha" na hierarquia |
| Aresta   | A conexão estre dois nós (pai e filho)                         |
| Rótulo   | Cada nó poderá ser associado a um identificar (rótulo)         |

Também existem alguma medidas importantes:

- **Nível ou profundidade:** é a distância de um nó até a raiz
- **Altura:** é o comprimento do caminho mais longo da raiz até qualquer nó. A altura de uma árvore vazia é -1, e 0 com somente a raiz

#### Árvores binárias

Uma árvore binária é uma estrutura de dados onde cada nó possui, no máximo, dois filhos. Esses filhos são convencionalmente chamados de Filho Esquerdo e Filho Direito. Ou seja, podemos dizer que toda árvore binária com **n** nós possui exatamente n+1 subárvores esquerdas e direitas.


##### Tipos especiais de árvores binárias

A classificação é baseada na distribuição dos nós nos níveis (altura) da árvore.

###### Árvore Binária Cheia

Uma árvore é **cheia** ou estritamente **binária** se todo nó tem zero ou dois filhos. Nenhum nó pode ter apenas um filho.


###### Árvore Binária Completa

Uma árvore é **completa** se:

1. Todos os níveis, exceto possivelmente o único, estão completamente preenchidos.
2. O último nível deve ser preenchido da esquerda para a direita

**Propriedade chave:** permite uma representação eficiente usando um array, onde o filho esquerdo no índice *i* está em 2*i*+1 e o filho direito em 2*i*+2.


###### Árvore Binária Perfeita

Uma árvore é **perfeita** se ela é estritamente **binária** e **completa** ao mesmo tempo.

Propriedade chave: todos os nós folhas estão no mesmo nível e todos os nós internos têm exatamente dois filhos.

Neste caso, o total de número de nós pode ser definido como:$$\text{Total de Nós} = 2^{H+1} - 1$$
Resumindo: 

| **Tipo**            | **Descrição**                                  | **Regra de Filhos**                               |
| ------------------- | ---------------------------------------------- | ------------------------------------------------- |
| **Cheia / Estrita** | Todo nó interno tem 2 filhos.                  | $\text{0 ou 2 filhos}$                            |
| **Completa**        | Níveis preenchidos da esquerda para a direita. | $\text{Pode ter 1 ou 2 filhos}$                   |
| **Perfeita**        | Estritamente Binária E Completa.               | $\text{2 filhos, todas as folhas no mesmo nível}$ |

#### Percurso em árvores binárias

Podemos definir percurso sendo uma visita sistemática a cada um de seus nós. Existem três métodos principais para percorrer uma árvore binária: **PreOrder**, **InOrder** e **PostOrder**. Neste caso, usaremos a primeira.

##### Pré-Ordem

**Ordem:** Raiz $\to$ Esquerda $\to$ Direita

Visita o nós antes de seus filhos. A sua principal utilidade é copiar ou serializar a árvore, além de também poder criar uma representação de prefixo.

**Exemplo:**

```
	A (Raiz)
   / \ 
   B  C
  / \ 
  D  E
```

Sequência: D $\to$ B $\to$ E $\to$ A $\to$ C


##### Em Ordem

**Ordem:** Esquerda $\to$ Raiz $\to$ Direita

Visita o nó entre a visita aos filhos. A sua principal utilidade é imprimir os nós de uma árvore binária de busca (BST) em ordem crescente.

**Exemplo:**

```
	A (Raiz)
   / \ 
   B  C
  / \ 
  D  E
```

Sequência: A $\to$ B $\to$ D $\to$ E $\to$ C


##### Pós-Ordem

**Ordem:** Esquerda $\to$ Direita $\to$ Raiz

Visita o nó após a visita aos filhos. Sua principal utilidade é excluir ou liberar nós da árvore, além de criar uma representação de pós-fixo (notação polonesa inversa).

**Exemplo:**

```
	A (Raiz)
   / \ 
   B  C
  / \ 
  D  E
```

Sequência: D $\to$ E $\to$ B $\to$ C $\to$ A

#### Conversão de uma floresta

Uma floresta é simplesmente um conjunto de árvores disjuntas (que não se tocam). A solução nesse caso é usar uma técnica conhecida como Primeiro Filho - Próximo Irmão (First Child – Next Sibling).
