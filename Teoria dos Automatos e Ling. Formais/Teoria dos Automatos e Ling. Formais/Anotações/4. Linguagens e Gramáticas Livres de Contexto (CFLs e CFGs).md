
#### Linguagens Livres de Contexto (CFLs)

Uma linguagem livre de contexto **(CFL)** é uma linguagem formal que pode ser gerada por uma gramática livre de contexto **(CFG)**.

Características principais:

- Linguagens formais: São conjuntos de cadeias formadas a partir de um alfabeto finito

- Livre de contexto: a gramática que gera essas linguagens tem a restrição de que, em suas regras de produção, o lado esquerdo de cada regra é sempre uma única variável (não pode ser uma combinação de variáveis e terminais).

Exemplos:

- A linguagem de todas as expressões aritméticas com números e operações, como:
	- `(a + b) * c`
	- `(x + y) * (a + b)`

- A linguagem de pares de parênteses balanceados, como:
	- `()`
	- `(())`
	- `(()())`


##### Propriedades de Linguagens Livres de Contexto

**Fechamento:**

- As linguagens livres de contexto são fechadas sob as operações de união, concatenação e estrela de Kleene.
- Não são fechadas sob interseção ou diferença, o que limita algumas operações em relação a outras classes

**Reconhecimento de CFLs:**

- As linguagens livres de contexto podem ser reconhecidas por autômatos de pilha, que são máquinas de estados com uma pilha adicional para armazenar informações durante a computação.

**Ambiguidade:**

- Uma gráfica ambígua é aquela que pode gerar uma cadeia de diferentes maneiras. Ou seja, existem diferentes árvores de derivação ou ordens de aplicação de regras para a mesma cadeia.
- Exemplo:
	- $S → S + S$
	- $S → S x S$
	- A expressão $3 + 2 * 5$ pode ser interpretada de duas formas:
		- $(3 + 2) * 5$
		- $3 + (2 * 5)$

#### Gramáticas Livres de Contexto (CFGs)

Uma gramática livre de contexto (CFG) é uma gramática formal que define uma linguagem livre de contexto. Ela é composta por um conjunto de regras de produção, onde cada regra tem a forma:

$A -> γ$

- A é uma variável (ou símbolo não terminal)
- γ é uma string composta de variáveis e terminais (que pode ser vazia)
- As variáveis representam não terminais e os terminais representam símbolos da linguagem.

Componentes de uma CFG:

1. **Variáveis (não terminais):** símbolos usados para gerar a linguagem. São substituídos por outras variáveis ou terminais
2. **Terminais:** símbolos da linguagem que aparecem na cadeia final. Não podem ser substituídos por outras variáveis
3. **Regras de produção:** definem como as variáveis podem ser substituídas por outras variáveis ou terminais
4. **Símbolo inicial:** Uma variável de onde o processo de derivação começa

Exemplo de CFG:

	Para construir uma gramátca que gera a linguagem de parênteses balanceados, como mencionamos antes, temos:

Gramática:

- Variáveis: $S$ (símbolo inicial)
- Terminais: ( e )
- Regras de produção: 
	- $S$ -> ($S$)$S$
	- $S$ -> ϵ (onde ϵ é a cadeia vazia)
- Símbolo inicial: $S$

Exemplos de derivação usando essa CFG:

1. Para a cadeia `()`, temos:
	- $S ⇒ (S)S ⇒ ()S ⇒ ()$  (derivação completa)

2. Para a cadeia `()()`, temos:
	- $S ⇒ (S)S ⇒ ()SS ⇒ ()()$

#### Diferenças entre Linguagens Regulares e Linguagens Livres de Contexto

**Linguagens Regulares** podem ser reconhecidas por autômatos finitos. São mais simples e menos poderosas. São uma subclasse das linguagens livres de contexto.

Já as **Linguagens Livres de Contexto** podem ser reconhecidas por autômatos de pilha, sendo mais poderosas que as linguagens regulares, pois permitem expressões mais complexas, como a de parênteses balanceados.

___

#### Aplicações de CFLs e CFGs

- **Compiladores:** As gramáticas livres de contexto são fundamentais na definição da sintaxe de linguagens de programação

- **Analisadores sintáticos (Parseadores):** Utilizam CFGs para reconhecer a estrutura sintática de programas, expressões matemáticas, etc.

- **Processamento de Linguagem Natural (PLN):** Muitas das regras que descrevem a gramática de uma linguagem natural podem ser modeladas com CFGs, como frases com estrutura de sujeito, verbo, objeto.
